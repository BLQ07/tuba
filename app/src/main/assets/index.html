1. Введение

Приложение «MathCad» для Android — мощный инструмент для математических расчётов, обработки данных и взаимодействия с микроконтроллерами ESP32. Оно сочетает в себе:

  • Редактор выражений с кастомной клавиатурой (математические функции, матрицы, операторы).
  • Систему листов – многострочные скрипты с присваиваниями, условиями (if/else/endif), циклами (for, while) и функцией print() для отправки данных на ESP.
  • Управление постоянными и временными константами.
  • Таблицы данных – удобное хранение параметров; загрузка строк таблицы как временных констант.
  • Полную матричную алгебру (определитель, обратная матрица, решение систем, векторные операции и т.д.).
  • Автоматическое подключение к ESP32, периодический опрос данных, управление переменными через слайдер.
  • Построение графиков (временные ряды или зависимость одной переменной от другой) с экспортом в CSV.
  • Настройку темы (светлая/тёмная) и цвета акцента.

2. Установка и настройка

2.1 Разрешения

Приложение запрашивает следующие разрешения (все они указаны в AndroidManifest.xml):

  • INTERNET – для связи с ESP.
  • ACCESS_NETWORK_STATE, ACCESS_WIFI_STATE, CHANGE_WIFI_STATE – для автоматического подключения к сети ESP.
  • WRITE_EXTERNAL_STORAGE – для сохранения CSV-файлов (только до Android 10).
  • ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION – для сканирования Wi-Fi (требуется на Android 6+).

2.2 Подготовка ESP32

ESP32 должен быть настроен как точка доступа с фиксированным IP 192.168.4.1. На нём должен работать HTTP-сервер, отвечающий на запросы:

  • GET /data – возвращает JSON со всеми переменными (например, {"x":10.5, "y":20.0}).
  • GET /set?var=имя&value=значение – устанавливает переменную и возвращает обновлённый JSON.

Рекомендуется использовать библиотеку ESPDataServer (входит в проект). SSID и пароль точки доступа по умолчанию: MathCad_ESP / 12345678. Их можно изменить в настройках приложения.

2.3 Подключение к ESP

  1. Включите ESP (питание).
  2. На главном экране приложения переключите переключатель ESP (в правой части тулбара).
  3. Приложение автоматически попытается подключиться к сети MathCad_ESP. Если сеть не найдена, появится инструкция по ручному подключению.
  4. После успешного подключения нажмите кнопку Play (вместо Stop) для запуска периодического опроса данных.

3. Главный экран

На главном экране расположены основные элементы управления:

  • Поле ввода выражений – многострочное поле для ввода выражений (если не выбран лист).
  • Кастомная клавиатура – содержит кнопки математических функций, операторов, матричных функций, служебных кнопок (Const, ←, пробел, перевод строки).
  • Кнопка «Вычислить всё» – вычисляет текущий лист (если выбран) или выражение из поля ввода.
  • Спиннер листов – выбор активного листа для вычислений.
  • Спиннер переменных – список всех доступных переменных (постоянные константы, данные ESP, временные константы). При выборе переменной появляется слайдер.
  • Слайдер (SeekBar) – изменяет значение выбранной переменной. При отпускании ползунка новое значение отправляется на ESP и сохраняется в соответствующую константу.
  • Область результатов – отображает результаты вычислений листа, сообщения об ошибках, подтверждения отправки print().
  • Блоки постоянных и временных констант – горизонтальные списки текущих значений.
  • Таблицы данных – спиннер таблиц, спиннер строк, кнопка «Добавить как константу».
  • Кнопка «График» – открывает экран построения графиков.
  • Кнопка «Play/Stop» – запуск/остановка опроса ESP.
  • Кнопка «Очистить временные константы» – удаляет все временные константы.

4. Константы

4.1 Постоянные константы

Сохраняются в памяти телефона (SharedPreferences) и не теряются при перезапуске. Каждая постоянная константа имеет имя, значение, минимальное и максимальное значение (для слайдера).

Добавление: нажмите кнопку Const на клавиатуре, введите имя и значение, затем задайте диапазон.

Редактирование: боковое меню → «Константы». Здесь можно изменять значения (автосохранение при потере фокуса), удалять константы.

4.2 Временные константы

Действуют только в текущей сессии. Создаются:

  • При выполнении присваивания в листе (a := 5).
  • При загрузке строки из таблицы (кнопка «Добавить как константу»).

Временные константы переопределяют постоянные с тем же именем (но не данные ESP). Их можно очистить кнопкой Очистить временные константы.

5. Таблицы данных

Таблицы позволяют хранить наборы параметров и быстро загружать их во временные константы.

5.1 Создание и редактирование

  • Боковое меню → «Таблицы».
  • Нажмите «Создать таблицу», введите имя и заголовки столбцов через запятую (обязательно должны быть столбцы name и value, остальные – произвольные).
  • В списке таблиц выберите созданную, откроется редактор.
  • Добавляйте строки кнопкой «+ Добавить строку». Первый столбец – ID автоматический, остальные – редактируемые ячейки.
  • Удалить строку можно только удалив всю таблицу (кнопка удаления в списке таблиц).

5.2 Использование на главном экране

  1. Выберите таблицу в спиннере «Таблицы данных».
  2. Выберите строку по имени (столбец name).
  3. Нажмите Добавить как константу. Все числовые значения из выбранной строки станут временными константами с именами столбцов.

6. Листы (скрипты)

Листы – многострочные текстовые файлы, содержащие последовательность инструкций. Редактор листа использует WebView с LaTeX-рендерингом и ту же кастомную клавиатуру.

6.1 Создание и редактирование

  • Боковое меню → «Листы».
  • Нажмите «Создать лист», введите имя и содержимое.
  • Для редактирования выберите лист из списка.
  • Сохраните изменения.

6.2 Синтаксис

Присваивание

a := 10
b := a * 2

Переменная a становится временной константой.

Функция print()

  • print(x) – отправляет текущее значение переменной x на ESP.
  • print(x, y*2) – вычисляет y*2 и отправляет результат на ESP под именем x.

Выражения

Просто строка (например, sin(x)+cos(y)) вычисляется; результат сохраняется в специальной переменной result.

Условный оператор if

if x < 10:
    print(x)
else:
    print(x, x*2)
endif

Условие может содержать операторы сравнения: <, >, <=, >=, ==, != (они автоматически заменяются на функции lt, gt, le, ge, eq, neq).

Цикл for

for i from 0 to 10:
    a := a + i
    print(a)
end

Переменная цикла доступна только внутри тела; изменяется от начального до конечного значения включительно с шагом 1.

Цикл while

i := 0
while i < 10:
    i := i + 1
    print(i)
end

Матричные функции в листах

A := matrix(2,2,1,2,3,4)
d := det(A)
print(d)
B := inv(A)
x := B[0][1]

6.3 Примеры листов

Пример 1: Простые вычисления и отправка

a := 10
b := 20
c := a + b
print(c)

Пример 2: Цикл и условный оператор

x := 0
while x < 10:
    x := x + 1
    if x > 5:
        print(x, x*2)
    else:
        print(x)
    endif
end

Пример 3: Работа с матрицами

A := matrix(2,2,1,2,3,4)
detA := det(A)
print(detA)
B := inv(A)
C := multiply(A, B)
print(C[0][0])

7. Матрицы и векторы

7.1 Создание матриц

  • matrix(rows, cols, v1, v2, ...) – создаёт матрицу rows×cols из перечисленных значений (по строкам).
  • eye(n) – единичная матрица n×n.
  • zeros(rows, cols) – нулевая матрица.
  • ones(rows, cols) – матрица из единиц.

7.2 Основные операции

7.3 Доступ к элементам

Используйте A[i][j], где i – номер строки, j – номер столбца (нумерация с 0).

A := matrix(2,2,1,2,3,4)
x := A[0][1]   // x = 2

7.4 Хранение матриц

Матрицы, созданные в листах, не сохраняются автоматически. Для постоянного хранения используйте Боковое меню → Матрицы. Там можно создать матрицу, указав размер и значения. Постоянные матрицы доступны в листах по имени.

8. Подключение к ESP32

8.1 Настройка ESP

ESP должен работать как точка доступа (AP) с фиксированным IP 192.168.4.1. Прошивка должна отвечать на /data и /set (см. раздел 2.2).

8.2 Настройка приложения

  • SSID и пароль ESP можно изменить в Боковое меню → Настройки.
  • Интервал опроса – как часто приложение будет запрашивать /data (мс).

8.3 Управление переменными

  • При выборе переменной в спиннере и перемещении слайдера новое значение отправляется на ESP через /set и сохраняется в константу (постоянную или временную).
  • В листах для отправки используйте print(имя) или print(имя, выражение).

9. Графики

9.1 Построение

  1. Остановите опрос ESP (кнопка Stop). Появится кнопка График, нажмите её.
  2. Выберите ось X и ось Y из списка переменных, для которых есть накопленные данные.
  3. Нажмите Построить.
    • Если выбрана ось X = «время», будет построен график изменения Y во времени.
    • Если выбрана другая переменная, будет построен график зависимости Y от X (данные синхронизируются по времени; точки выводятся только для моментов, когда есть значения обеих переменных).

9.2 Экспорт в CSV

  • В окне графика нажмите Экспорт CSV.
  • Файл сохранится в папку приложения. После сохранения появится диалог для отправки (поделиться).

10. Настройки

Боковое меню → Настройки:

  • ESP настройки – SSID и пароль точки доступа ESP (по умолчанию MathCad_ESP / 12345678).
  • Тема – светлая/тёмная/системная; цвет акцента (фиолетовый, красный, синий, зелёный, оранжевый).
  • Интервал опроса ESP – частота запросов к ESP (мс).

После изменения темы рекомендуется перезапустить приложение.

11. Часто задаваемые вопросы и устранение неполадок

Ошибка «Cleartext HTTP traffic not permitted»
Решение: добавьте в AndroidManifest.xml атрибут android:usesCleartextTraffic="true" в тег <application>.
Ошибка «Unknown function or variable 'if'»
Убедитесь, что лист сохранён, и при вычислении выбран этот лист. Проверьте синтаксис: if условие: (двоеточие обязательно) и endif на отдельной строке.
Ошибка «Operator is unknown for token» в условии
Парсер не понимает операторы сравнения. Используйте функции lt(), gt(), le(), ge(), eq(), neq() напрямую, например lt(x,10).
Не отображаются строки таблицы в спиннере
Убедитесь, что таблица создана с заголовком name, и в ней есть строки. После добавления строк закройте редактор таблицы и снова выберите таблицу на главном экране.
График не отображает все точки
Убедитесь, что в MainActivity и GraphActivity данные получаются через MyApplication.getDataCollector() (единый экземпляр).
Ошибка «Connection reset» при отправке переменной
ESP может не успевать обрабатывать частые запросы. В приложении добавлена задержка (debounce) 200 мс, но если проблема сохраняется, увеличьте задержку в коде или используйте очередь с повторными попытками.
Имена переменных
Рекомендуется использовать латиницу, цифры, подчёркивание, начинать с буквы. Избегайте имён pi, e, sin, cos и т.п. (зарезервированы).
