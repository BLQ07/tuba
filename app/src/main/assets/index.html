🚀 1. Первый запуск и интерфейс

Приложение разделено на 5 удобных вкладок: Управление, Результаты, Листы, График, Настройки. Все данные (константы, листы, таблицы) сохраняются локально.

🕹️ Управление (вкладка 1)

  • Автоподключение к ESP (руководство по Wi-Fi)
  • Кнопка Play/Stop для опроса датчиков
  • Выбор переменной и слайдер управления
  • Список всех переменных ESP с возможностью отправки

📋 Результаты (вкладка 2)

  • Вывод вычислений листа
  • Встроенный лог (переключение через спиннер)
  • Копирование результатов долгим нажатием

📑 Листы (вкладка 3)

  • Создание / редактирование / переименование / удаление листов
  • Поддержка циклов for, while, условий if
  • Функция print() для отправки данных на ESP

📈 График (вкладка 4)

  • Выбор осей X и Y из любых переменных
  • Экспорт CSV, масштабирование жестами
  • Синхронизация по времени или зависимость «переменная от переменной»

⚙️ Настройки (вкладка 5)

  • SSID / пароль ESP
  • Интервал опроса (мс)
  • Светлая/тёмная тема, цвет акцента
💡 Совет: Для быстрого доступа к константам и таблицам используйте боковое меню (три полоски в тулбаре). Там же — экспорт всех данных в CSV.

📄 2. Листы (скрипты)

Листы — последовательность команд, присваиваний и управляющих конструкций. Редактор листа использует WebView + LaTeX-рендеринг и кастомную клавиатуру.

📌 Синтаксис

# Присваивание
a := 10
b := a * 2

# Выражение (результат сохраняется в переменную result)
sin(a) + cos(b)

# Отправка на ESP
print(a)
print(temperature, a + 273.15)

# Условный оператор
if x > 5:
    print(x)
else:
    print(0)
endif

# Цикл for
for i from 0 to 10:
    sum := sum + i
end

# Цикл while
i := 0
while i < 10:
    i := i + 1
    print(i)
end

🖨️ print() — отправка на ESP

  • print(имя) – отправляет текущее значение переменной (ищет в постоянных, временных, данных ESP).
  • print(имя, выражение) – вычисляет выражение и отправляет результат под указанным именем.

Пример: print(duty, pwm * 100 / 255)

📂 Управление листами: на вкладке «Листы» есть кнопки: Создать, Редактировать, Переименовать, Удалить. При сохранении лист автоматически пересчитывается.

🔢 3. Константы

Постоянные константы хранятся в SharedPreferences и не исчезают после перезапуска. Временные константы создаются в листах через := или загрузкой из таблиц.

📌 Постоянные

  • Добавление: кнопка Const на клавиатуре → имя + значение + min/max.
  • Редактирование: боковое меню → Константы.
  • Используются во всех выражениях, видны на вкладке «Листы».

⏱️ Временные

  • Живут только во время сеанса.
  • Переопределяют постоянные с тем же именем.
  • Очищаются кнопкой «Очистить временные константы» на вкладке «Управление».
⚠️ Важно: избегайте имён pi, e, sin, cos – они зарезервированы. Имена переменных: латиница, цифры, подчёркивание, не начинаются с цифры.

📊 4. Таблицы данных

Таблицы — удобный способ хранить параметры (например, калибровочные данные). Каждая строка может быть загружена как набор временных констант.

  • Создание таблицы: боковое меню → Таблицы → «Создать таблицу».
  • Обязательные колонки: name (название строки) и value (числовое значение), остальные колонки произвольные.
  • Редактор таблицы позволяет добавлять строки, редактировать ячейки (автосохранение).
  • Использование на главном экране: выберите таблицу → выберите строку по имени → нажмите «Добавить как константу» → все числовые поля строки станут временными константами (имена = заголовки колонок).

Это позволяет быстро подставить набор параметров, например: kp, ki, kd из одной строки таблицы.

🧮 5. Матрицы и векторы

Приложение поддерживает полноценную матричную алгебру. Матрицы можно создавать через функции прямо в листах или через менеджер матриц (боковое меню).

📌 Создание

A := matrix(2,2,1,2,3,4)
I := eye(3)
Z := zeros(2,2)
v := matrix(1,3,1,2,3)

🔧 Операции

  • det(A) – определитель
  • inv(A) – обратная матрица
  • transpose(A) – транспонирование
  • add(A,B), multiply(A,B)
  • solve(A,b) – решение СЛАУ
  • norm(v), dot(u,v), cross(u,v)

Доступ к элементу: A[0][1] (нумерация с 0). Матрицы, созданные в листах, временные; постоянные матрицы хранятся в менеджере матриц.

📡 6. Подключение к ESP32

ESP работает как точка доступа (AP) с фиксированным IP 192.168.4.1. Приложение может управлять переменными и получать данные через HTTP GET.

🔌 Настройка ESP (прошивка)

  • ESP должен отвечать на GET /data → JSON всех переменных: {"temperature":25.5,"pwm":128}
  • ESP должен отвечать на GET /set?var=имя&value=значение → установка переменной и возврат обновлённого JSON.
  • Рекомендуется использовать библиотеку ESPDataServer (предоставлена в проекте).

📲 Работа в приложении

  • Переключатель ESP на тулбаре включает автоподключение.
  • При включении отображается диалог с инструкцией подключения к сети MathCad_ESP (SSID и пароль настраиваются).
  • После подключения нажмите Play — начнётся периодический опрос /data.
  • Слайдер на вкладке «Управление» отправляет новое значение на ESP через /set и сохраняет в постоянную константу (если такая есть).
  • Все переменные ESP отображаются внизу вкладки «Управление» — их можно отправить повторно кнопкой «Отправить».
💡 Совет: интервал опроса регулируется в настройках (мс). При высокой частоте ESP может не успевать — увеличьте интервал до 1000–2000 мс.

📈 7. Графики

Все полученные данные (ESP, присваивания, вычисления) автоматически сохраняются в DataCollector и доступны для построения графиков.

  • На вкладке «График» выберите ось X и ось Y. Если X = «время» — график изменения переменной во времени. Если X — другая переменная, строится scatter‑график (синхронизация по времени).
  • Кнопка «Экспорт CSV» сохраняет текущий график в файл (доступен для отправки).
  • График можно масштабировать жестами (сведение/разведение пальцев).
📊 Примечание: данные для графика накапливаются с момента запуска опроса или изменения слайдера. Очистить историю можно только перезапуском приложения (или через кнопку очистки в коде).

❓ 8. Частые вопросы и решения

❌ Ошибка «addNetwork returned -1»

Это означает, что автоматическое подключение к Wi-Fi ESP не удалось. Приложение использует безопасный метод: при включении автоподключения показывается диалог с инструкцией. Подключитесь вручную через настройки телефона к сети MathCad_ESP, затем вернитесь в приложение и включите опрос.

📡 Нет данных от ESP

Проверьте, что ESP создаёт сеть, SSID и пароль совпадают с настройками (меню → Настройки). Убедитесь, что IP телефона в сети ESP: обычно 192.168.4.x. Также проверьте, что ESP отвечает на запрос /data (можно открыть в браузере телефона).

📉 График не отображает все точки

Убедитесь, что используется единый DataCollector (доступен через MyApplication.getDataCollector()). После каждого изменения слайдера или получения данных от ESP точки должны добавляться. Проверьте логи с фильтром DataCollector.

🖨️ print() не отправляет значение

Проверьте, что имя переменной существует (постоянная константа, временная или данные ESP). Для отладки используйте вкладку «Результаты» в режиме «Лог» — там будут видны ошибки и сообщения.

⚡ Матричные функции не работают

Убедитесь, что вы используете правильный синтаксис: det(A), inv(A), multiply(A,B). Для доступа к элементу: A[0][1]. Матрицы должны быть созданы заранее.
